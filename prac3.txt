

## 1. Цель установки сервера

**Определение типа сервера:**

| Тип сервера       | Назначение                                                                                     |
| ----------------- | ---------------------------------------------------------------------------------------------- |
| Веб-сервер        | Обеспечивает доступ пользователей к веб-приложениям и веб-сайтам через HTTP/HTTPS.             |
| Сервер баз данных | Централизованное хранение, обработка и предоставление структурированных данных для приложений. |
| Файловый сервер   | Организация централизованного хранения и обмена файлами для пользователей сети.                |

**Потребности пользователей:**

* Обучающийся/сотрудник: быстрый и надежный доступ к информации.
* Администратор: возможность управлять сервером, настраивать права доступа и обеспечивать безопасность.
* Разработчики: доступ к веб-сервисам, базам данных и совместным файлам.

---

## 2. Описание среды установки

| Среда              | Рекомендации                                                                                                                                     |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| Виртуальная машина | Используется для тестирования и быстрого развёртывания серверов. Подходит для разработки и учебных целей.                                        |
| Физический сервер  | Используется для производственных сред с высокой нагрузкой. Обеспечивает стабильность и максимальную производительность.                         |
| Облачное решение   | AWS, Azure, Google Cloud. Подходит для масштабируемых и удалённых инфраструктур, с возможностью резервного копирования и быстрого развёртывания. |

---

## 3. Требования к серверу

### 3.1 Аппаратные требования

| Параметр              | Минимальные значения                                      |
| --------------------- | --------------------------------------------------------- |
| CPU                   | 2 ядра, 2.0 GHz                                           |
| RAM                   | 4 GB                                                      |
| Дисковое пространство | 50 GB (в зависимости от типа сервера, для БД — от 100 GB) |
| Сетевые интерфейсы    | 1 сетевой адаптер с пропускной способностью 1 Gbps        |

### 3.2 Программные требования

| Параметр                 | Рекомендации                                                                                                       |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------ |
| ОС для Linux             | Ubuntu Server 22.04 LTS / CentOS 9                                                                                 |
| ОС для Windows Server    | Windows Server 2022                                                                                                |
| Необходимые пакеты Linux | `apache2` или `nginx` (для веб-сервера), `mysql-server` или `postgresql` (для БД), `samba` (для файлового сервера) |
| Дополнительно            | OpenSSH, curl, ufw (Linux)                                                                                         |

### 3.3 Сетевые требования

* IP-адрес: статический для серверов в сети (рекомендуется), динамический — для тестовой среды.
* Доступ к интернету: необходим для обновлений и установки пакетов.
* Доступ в локальную сеть: обязательный для взаимодействия пользователей с сервером.

## 4. Пошаговые инструкции по установке

### 4.1 Подготовка сервера

**Linux (Ubuntu Server)**

1. Создать виртуальную машину (VM) или подготовить физический сервер.
2. Подключить ISO-образ Ubuntu Server 22.04 LTS.
3. Запустить установку и выбрать язык.
4. Настроить сетевой интерфейс (статический IP для продакшн-серверов).
5. Установить минимальный набор пакетов, включая OpenSSH.

**Windows Server 2022**

1. Подготовить сервер (VM или физический).
2. Подключить ISO-образ Windows Server 2022.
3. Следовать мастеру установки: выбрать язык, версию и место установки.
4. Настроить сетевой интерфейс (статический IP).
5. Установить роли сервера (IIS для веб-сервера, File Server для файлового сервера, SQL Server для базы данных).

---

### 4.2 Установка веб-сервера

**Linux:**

```bash
sudo apt update
sudo apt install apache2 -y
sudo systemctl enable apache2
sudo systemctl start apache2
```

**Windows:**

1. Открыть "Server Manager" → "Add roles and features".
2. Выбрать "Web Server (IIS)" → "Next" → "Install".
3. Запустить IIS и проверить доступ через браузер (`http://<IP>`).

---

### 4.3 Установка сервера баз данных

**Linux:**

```bash
sudo apt install mysql-server -y
sudo systemctl enable mysql
sudo systemctl start mysql
sudo mysql_secure_installation
```

**Windows:**

1. Установить SQL Server через официальный инсталлятор.
2. Настроить аутентификацию и создать базу данных.

---

### 4.4 Установка файлового сервера

**Linux (Samba):**

```bash
sudo apt install samba -y
sudo nano /etc/samba/smb.conf
# Настроить общий доступ к папкам
sudo systemctl restart smbd
```

**Windows:**

1. Настроить File Server через "Server Manager".
2. Создать общие папки и назначить права доступа.

---

## 5. Тестирование после установки

1. Проверка доступности по SSH (Linux):

```bash
ssh user@<IP>
```

2. Проверка конфигурации сети: `ping`, `ifconfig`/`ip a`, `nslookup`.
3. Проверка работы веб-сервера: открыть браузер `http://<IP>` и убедиться, что отображается стартовая страница.
4. Проверка работы базы данных: подключение через `mysql -u root -p`.
5. Проверка файлового сервера: доступ к общим папкам с другого устройства.
6. Проверка сохранности после перезагрузки: `sudo reboot` → повторить шаги проверки.

**Ожидаемые результаты:** сервер доступен, службы запущены, конфигурация сохранена.

## 6. Рекомендации по безопасности

1. Установить все обновления системы:

**Linux:**

```bash
sudo apt update && sudo apt upgrade -y
```

**Windows:**

* Использовать Windows Update.

2. Настроить брандмауэр:

**Linux (UFW):**

```bash
sudo ufw allow OpenSSH
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
```

**Windows:**

* Настроить правила входящего трафика через Windows Firewall.

3. Ограничение доступа пользователей:

* Создать отдельные учетные записи для администраторов и обычных пользователей.
* Назначить минимальные права доступа.

4. Интеграция с внешними сервисами:

* Подключение к LDAP, Active Directory или облачным сервисам (по необходимости).
